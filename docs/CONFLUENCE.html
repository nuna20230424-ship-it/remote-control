§1기획

목표: 사업자·단말에 종속되지 않고, STB UI를 스스로 탐색·학습해 자연어 목표를 달성하는 리모컨 제어 에이전트.

핵심 유스케이스

  • UC-1 학습: 리모컨 키를 눌러가며 화면 전이를 관찰, 네비게이션 맵을 구축.
  • UC-2 맵 조회: 학습된 상태/전이/커버리지를 조회.
  • UC-3 목표 실행: "넷플릭스 켜줘" 같은 목표 → 최단 버튼 경로 계획·실행, 어긋나면 재계획.

스코프: (In) mock 만으로 전 구간 완결 · 실 STB 교체식 어댑터 · REST/CLI/MCP. (Out, 후속) IR 코드 캡처(ir-mcp /learn 영역) · 사업자별 화면 라벨 온톨로지.

§2설계

파이프라인: 학습(Learner) → 맵(NavGraph) → 에이전트(Planner+Executor).

교체 가능한 어댑터 (코어 엔진 변경 0줄):

  • RemoteDriver — MockRemoteDriver(개발) / RemoteMcpClient(실: ir-mcp + capture-mcp)
  • ScreenSense — MockScreenSense(개발) / DetectionMcpScreenSense(실: detection-mcp/VLM)

실 STB 스택 = 3 서비스

서비스엔드포인트remotectl 처리
ir-mcp :8002POST /send {codeset, key}키맵(DPAD_*/VOLUME_*/숫자/앱), repeat→N회, reset→HOME 폴백, /codesets→실 키집합
capture-mcp :8001POST /capture → MP4ffmpeg로 마지막 프레임 PNG 추출
detection-mcp :8103POST /check/screen → verdict+descriptionQA 판정기 → description(VLM 서술)을 로컬 합성해 상태 식별

영속화: 네비게이션 맵 = JSON(networkx) · 키이벤트↔화면 매핑 = 별도 SQLite DB(navmap과 분리).

#5구현 기능 개요 페이지(docs/features.html)

추가 기능(#4)

  • 동시 학습: 스텝마다 (from,key)→to 매핑과 화면 LLM 분석을 함께 별도 DB에 기록.
  • 오류 자가치유: 전송 실패→재전송, 판정 실패/저신뢰/비결정 전이→detection-mcp VLM 재판정으로 재매핑, 소진 시 건너뜀.
  • 자동 반복: 목표 커버리지 도달/무진전/상한 종료 + 미커버 키·상태 리포트.
커버리지의 정의 — "발견된 각 화면에서 각 실 리모컨 키의 효과(전이)를 관측한 비율". IR 코드 캡처가 아니다. 100%는 미발견 상태·도달불가 키 때문에 항상 보장되지 않으므로 목표 커버리지를 파라미터로 받고 미달(미커버 키/상태)을 숨기지 않는다.

§5리스크 / 로드맵

  1. detection 서명 안정성 튜닝 — 화면 서명이 흔들리면 그래프 파편화로 커버리지가 무의미. 서버측 vision 프롬프트/키워드가 최대 레버.
  2. 실 STB 스모크 — 코드셋 실측 매칭(GET /codesets), capture 공유 볼륨 + ffmpeg. (사내망 실기기 필요)
  3. 커버리지 완전성 — 100%는 "발견 상태 기준". 상태 분류 전용 baseline 마련 시 description 합성 → 그 경로로 승격.

§6배포물 (git 없이 사용)

git 접근 없이도 배포된 wheel 파일만으로 설치·실행 가능. 상세는 docs/DISTRIBUTION.md.

pip install remotectl-0.1.0-py3-none-any.whl   # 배포 파일만으로 설치
remotectl serve                                 # mock 으로 즉시 실행
  • 대시보드 정적 자산이 wheel 에 포함 → 별도 파일 없이 대시보드가 뜬다.
  • mock 기본이라 사내망/실 STB 없이 학습→맵→목표실행→자동반복 전 구간 동작.
  • 재생성: bash scripts/build_dist.sh → dist/*.whl 을 팀에 전달.
