§4

동시 학습 · 별도 DB · 자가치유 · 자동 반복

리모컨 키와 화면을 한 패스에서 동시에 학습하고 LLM(VLM) 분석과 함께 별도 DB에 쌓으며, 오류는 자가치유하고, 목표 커버리지까지 자동 반복한다. 코어 경계(M5) 유지 — 엔진은 추상 훅만 안다.

동시 학습

engine/hooks.py

스텝마다 (from,key)→to 매핑 + 화면 LLM 분석을 함께 기록(StepObserver 훅).

별도 DB

store.py · SQLite

screens·key_screen_map·error_log. navmap(JSON)과 분리. REMOTECTL_KEYSCREEN_DB.

오류 자가치유

reconcile.py

전송실패→재전송, 판정실패/저신뢰/비결정→VLM 재판정(REOBSERVE) 재매핑, 소진→건너뜀.

자동 반복

autolearn.py

목표 커버리지 도달 / 무진전 / 상한 종료. 미커버 키·상태 리포트.

100% 는 미발견 상태·도달불가 키 때문에 항상 보장되지 않는다 → target_coverage 를 받고 uncovered_key_tokens/uncovered_states 를 리포트로 노출한다(silent 미달 방지). 커버리지 분모는 실 리모컨 키 전체(드라이버 available_keys(), POWER 등 안전 제외)이며, "발견된 상태 기준"임을 문서에 명시한다.
§5

실 STB 배선 — 3 서비스

사내 스택 실물 계약에 맞춰 배선. 제어·캡처·판정이 세 서비스로 분리되며, 두 곳은 단순 경로가 아닌 의미 변환을 어댑터가 흡수한다.

서비스엔드포인트배선 처리
ir-mcp
:8002		 POST /send {codeset, key} 키맵 DPAD_*/VOLUME_*/숫자/앱대문자 · repeat→N회 · reset→HOME 폴백 · /codesets→키집합
capture-mcp
:8001		 POST /capture → MP4 영상만 제공 → ffmpeg로 마지막 프레임 PNG 추출해 스틸로 사용
detection-mcp
:8103		 POST /check/screen → verdict + description QA 판정기이므로 description(VLM 서술)을 키워드/의미토큰으로 로컬 합성 → 상태 식별
